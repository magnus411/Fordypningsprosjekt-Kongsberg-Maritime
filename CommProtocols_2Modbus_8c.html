Sensor Data Handler System  1.0
This file implements the main entry point and configuration setup for the Kongsberg Maritime's sensor data handling system. The system is designed to collect and store sensor data with flexible and modular configuration capabilities.\n \n This system can be configured to handle multiple data handlers, each with their own thread group. The system is designed to be easily extensible and configurable through JSON configuration files.\n \n Changes to the code can be made to extend the system's capabilities, such as adding protocols. This system has built up a framework for adding new configurations later.

Parses a Modbus TCP frame into its components.

Parses a Modbus TCP frame.

Frame Structure:

| Field | Size | Description |
|---------------|---------|------------------------|
| Transaction ID| 2 bytes | Request identifier |
| Protocol ID | 2 bytes | Protocol (0 = Modbus) |
| Length | 2 bytes | Number of following bytes|
| Unit ID | 1 byte | Slave identifier |
| Function Code | 1 byte | Action code |
| DataLength | 1 byte | Length of data field |
| Data | n bytes | Payload data |

The function checks for frame consistency by verifying that the Length field matches DataLength + 3 (accounting for UnitID, FunctionCode, and ByteCount).

Parameters
FramePointer to the raw Modbus TCP frame
UnitIdPointer to store the parsed Unit ID
DataLengthPointer to store the data length
Returns
Pointer to the start of data section, NULL if parsing fails or frame is invalid
See also
https://en.wikipedia.org/wiki/Modbus#Public_function_codes
https://modbus.org/docs/Modbus_Application_Protocol_V1_1b.pdf
Note
Bytes are arranged in big-endian order (most significant byte first)
